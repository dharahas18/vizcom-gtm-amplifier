# Template — Product Launch

**signal_type =** `product_launch`.

Opener is aphoristic. Body is operational. Closer is "available now."

Channel-length / hashtag / emoji rules live in `core_rules.md`.

## `blog.md`

### H1 patterns
- `Meet {Feature}: {one-line benefit}`
- `Introducing {Feature}`
- `A render is only the beginning: Introducing {F1}, {F2} & {F3}`
- `Expand {capacity / capability} with {Feature}`

### Dek
What the feature is + what it does + who it's for.

### Body H2 outline

```
[Aphoristic one-line hook — "Design is a physical act." / "Great design rarely happens in a straight line."]

[1–2 sentences connecting the hook to product reality.]

## Why we built this
(Pain in the user's own voice — "We've heard the same thing from you: {pain quote}.")

## How {it works / they work together}
   ### H3 — {Verb-led benefit headline + feature name}. E.g., "Change one color, not the whole concept with Extract."
   ### H3 — {Verb-led benefit headline #2}.
   ### H3 — {Verb-led benefit headline #3}.

## Available now {— for all users / Pro users / etc.}

## (optional) What's next
```

### Closing CTAs
- `See these updates in action ➡️ [Watch the video]({video URL})`
- `Get Started ➡️ [Open Vizcom](https://app.vizcom.com)`

### Template-specific forbidden moves
- Adjective stacks ("powerful, intuitive, transformative").
- "Empowers designers to…" — replace with verb + benefit.
- Promising the future ("the future of design") without earning it.

## `linkedin.md` — opener formulas

Aphorism that connects to the feature:
- `A reference says more than a paragraph.` (Collections)
- `Design is a physical act.` (Extract/Try On/Export)
- `Some months you're deep in iteration. Others you're pushing renders.` (Credits)

Body: 2 short paragraphs (feature + constraint preserved; workflow it unlocks).

Closer (capability-summary line):
- `Build it once, use it across every project.`
- `The workflow stays connected, because the best ideas don't wait for the tools to catch up.`

## `x_thread.md` — opener formulas

**Single-tweet launch (preferred for small features):**
```
{Aphorism}. {Feature name(s)} {is / are} now live in Vizcom.
```

**Two-tweet pair (for multi-feature or significant launches):**

Tweet 1:
```
{Aphorism that frames the problem the feature solves}.

{Feature name(s)} {is / are} now live in Vizcom.
```

Tweet 2:
```
Read the full story: {blog URL}
```

## `geo_aeo.md`

```
**Q: What is {Feature} in Vizcom?**

A: {Feature} is Vizcom's {one-line description}, available {free / for Pro / for all users}.

[2–3 sentences: how it fits the workflow, specific limit/capacity (numbers!), how it interacts with other Vizcom features.]

**See also:** {canonical Vizcom blog post URL} • {docs.vizcom.com URL if relevant}
```

## `pr_pitch.md`

**Subject:** `Vizcom ships {Feature}`

**Three sentences:**
1. `Vizcom today launched {Feature}, {one-line description with a specific capacity number}.`
2. `The feature {names the workflow it improves} for {industry segment}, with {specific capability}.`
3. `Sophia Silver, Product Marketing Manager at Vizcom, is available for comment; full announcement at {blog URL}.`

**Contact:** Sophia Silver. (See `core_rules.md` Section 9.)

---

## `summary.md` notes

- Default contact: **Sophia Silver**.
- Sequence: blog T+0 / LinkedIn +15m / X +30m / Instagram +60m / PR parallel.
- Voice rule to double-check: no hype words. Product launches are where adjective creep is most likely.

## Worked example: Smart Dropper launch

**Input:**
> "We just shipped Smart Dropper. It absorbs the vibe of any reference image — point it at a retro Mac or your favorite album cover and it applies the style. Going live today. Render at /renders/smart_dropper_hero.png. Demo video: youtu.be/X."

**Expected outputs:**
- `blog.md` (~900 words): "Meet Smart Dropper: A reference becomes a style" → aphoristic hook → Why we built this → How it works (H3 dropper, H3 composite-back) → Available now → CTA.
- `linkedin.md`: hook "A reference says more than a prompt." + 2 paragraphs + capability-summary closer.
- `x_thread.md`: single tweet — "Your designs can look like anything now. Smart Dropper is now live in Vizcom." + demo video.
- `geo_aeo.md`, `pr_pitch.md`.
